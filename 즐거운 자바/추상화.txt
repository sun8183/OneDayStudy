추상화 : 중요한 것은 남기고, 불필요한 것은 제거한다. 

1. 객체 상황에 따라서 가져야할 정보는 달라진다. 

2. 비즈니스 영역( 도메인 영역 )에 맞도록 추상화를 해야한다. 

캡슐화 ( encapsuation )
 - 관련된 것을 잘 모아서 가지고 있는 것을 캡슐화 라고 말한다.
관련된 것을 잘 모아서 가지고 있을수록 응집도( cohesion ) 가 높다고 표현한다. 

EX) Math 클래스에 수학관련 클래스만 가져야 하는것 
 - 클래스 안에 서로 연관있는 속성과 기능들을 하나의 캡슐로 만들어 데이터를 외부로 부터 보호하는 것 

추상클래스 - 인스턴스가 될 수 없는 클래스 ( 추상클래스를 상속받는 자손이 인스턴스가 된다.)
abstract 키워드를 사용하여 클래스 정의, 선언만 되어 있으며 구현이 아직 되어 있지 않음

단, 추상클래스 타입으로 배열을 선언하여 자손 인스턴스를 생성하는 것은 가능하다.

1. 초기화 protected final

2. 실행 public 

3. 마무리 protected final
